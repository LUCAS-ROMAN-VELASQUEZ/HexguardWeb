Módulos de Hardening

Cada módulo es independiente y puede ejecutarse de forma aislada o en conjunto. Todos los cambios se registran en /var/log/hexguard.log.

// MOD-01
📦
Actualización del Sistema

Actualiza el índice de paquetes, aplica todos los upgrades de seguridad disponibles y elimina dependencias huérfanas.

--update  |  apt update + upgrade
// MOD-02
🔥
Firewall UFW

Instala y configura UFW con política deny-all en entrante. Abre únicamente los puertos SSH (22), HTTP (80) y HTTPS (443).

--firewall  |  ufw + reglas
// MOD-03
🔐
SSH Hardening

Modifica /etc/ssh/sshd_config: deshabilita login root, autenticación por contraseña, reduce MaxAuthTries a 3 y LoginGraceTime a 30s. Crea backup automático antes de modificar.

--ssh  |  sshd_config
// MOD-04
🛡
Fail2Ban

Instala y activa Fail2Ban para bloquear automáticamente IPs que superan el umbral de intentos de autenticación fallidos.

--fail2ban  |  jail SSH
// MOD-05
🔍
Escaneo de Rootkits

Ejecuta chkrootkit para detectar rootkits, backdoors y exploits locales conocidos en el sistema.

--scan  |  chkrootkit
// MOD-06
👤
Auditoría de Usuarios

Lista todos los usuarios del sistema, detecta cuentas con UID 0 (equivalentes a root), shells interactivas activas y el historial de los últimos 10 logins.

--audit  |  /etc/passwd
// MOD-07
⛔
Servicios Inseguros

Detecta y deshabilita servicios que transmiten datos en texto plano: telnet, rsh, rlogin, ftp, vsftpd. Solo actúa si están instalados.

--services  |  systemctl disable
// MOD-08
📁
Permisos Críticos

Verifica y corrige permisos de /etc/passwd (644), /etc/shadow (640), /root (700) y /etc/sudoers (440).

--perms  |  chmod + chown
// MOD-09
📋
Análisis de Logs SSH

Lee /var/log/auth.log (o /var/log/secure) y muestra los últimos 20 intentos fallidos y el top 10 de IPs atacantes.

--logs  |  auth.log
// MOD-10
⚡
Hardening Completo

Ejecuta los 10 módulos en secuencia óptima con confirmación previa. Genera un resumen final con inicio, fin y duración total.

--full  |  todos los módulos
// Inicio rápido

Instalación

HexGuard no tiene dependencias externas. Solo Python 3.8+ y privilegios root. Descarga el script y ejecútalo.

root@kali — bash — 80×24
root@kali:~# # Opción 1 — Descarga directa
root@kali:~# wget https://tudominio.com/hexguard.py
root@kali:~# # Opción 2 — Clonar repositorio
root@kali:~# git clone https://github.com/tuusuario/hexguard.git && cd hexguard
root@kali:~# # Dar permisos de ejecución
root@kali:~# chmod +x hexguard.py
root@kali:~# # Verificar instalación
root@kali:~# sudo python3 hexguard.py --version

Cómo Usarlo

HexGuard funciona en dos modos: menú interactivo (sin argumentos) o modo directo pasando flags.

ejemplos de uso
root@kali:~# sudo python3 hexguard.py
# Lanza el menú interactivo completo
root@kali:~# sudo python3 hexguard.py --full
# Hardening completo (todos los módulos)
root@kali:~# sudo python3 hexguard.py --firewall --ssh
# Solo firewall + SSH hardening
root@kali:~# sudo python3 hexguard.py --scan --logs
# Escanear rootkits + analizar logs SSH
root@kali:~# sudo python3 hexguard.py --audit --perms
# Auditoría usuarios + verificar permisos
root@kali:~# sudo python3 hexguard.py --full --no-banner
# Hardening completo sin banner (ideal en scripts)

Advertencias

⚠ Uso Exclusivamente Autorizado

HexGuard modifica configuraciones críticas del sistema. Úsalo únicamente en sistemas de tu propiedad o con autorización expresa por escrito. El uso no autorizado puede ser ilegal en tu jurisdicción.

⚠ Módulo SSH — Lee antes de aplicar

El módulo --ssh deshabilita la autenticación por contraseña. Si aún no tienes una clave SSH configurada en el servidor, perderás el acceso remoto. Configura tu clave pública antes de ejecutar este módulo. Se crea un backup automático en /etc/ssh/sshd_config.bak.

⚠ Hardening Completo — Ten consola de emergencia

El modo --full modifica simultáneamente firewall, SSH y permisos del sistema. Se recomienda ejecutarlo con acceso físico al servidor o desde una consola de emergencia / VNC, por si necesitas revertir cambios.

Restaurar SSH: Si necesitas revertir la configuración SSH:
sudo cp /etc/ssh/sshd_config.bak /etc/ssh/sshd_config && sudo systemctl restart ssh
